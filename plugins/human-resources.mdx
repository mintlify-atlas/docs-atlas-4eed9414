A people operations plugin for Cowork that helps with recruiting, onboarding, performance management, policy guidance, and compensation analysis. Works standalone or supercharged when you connect your HRIS, ATS, and other tools.

## Installation

```bash
claude plugins add knowledge-work-plugins/human-resources
```

### /draft-offer

Draft a complete offer letter for a new hire.

```bash
/draft-offer <role and level>
```

**What you need to provide:**
- Role and title
- Level (Junior, Mid, Senior, Staff, etc.)
- Location (affects comp and benefits)
- Compensation: base salary, equity, signing bonus
- Start date
- Hiring manager

**Example:**
```bash
/draft-offer Senior Software Engineer
```

**Output includes:**
- Complete compensation package breakdown
- Offer letter text
- Benefits summary
- Notes for hiring manager with comp band context

### /onboarding

Generate a comprehensive onboarding plan.

```bash
/onboarding
```

Provide the new hire's name, role, team, and start date. Get:
- Onboarding checklist
- First-week calendar
- Tool access list
- Buddy assignment template

### /performance-review

Structure a performance review process.

```bash
/performance-review
```

Get templates for:
- Self-assessments with competency-based prompts
- Manager review template
- Calibration preparation
- Feedback that's specific, actionable, and fair

### /policy-lookup

Find and explain company policies.

```bash
/policy-lookup <policy topic>
```

Or just ask naturally:
```
What's our parental leave policy?
```

The `employee-handbook` skill triggers automatically and searches your connected knowledge base.

### /comp-analysis

Analyze compensation data with market benchmarking.

```bash
/comp-analysis
```

Upload comp data or describe your bands. Get:
- Market comparisons
- Band placement analysis
- Recommendations for adjustments
- Equity refresh modeling

### /people-report

Generate workforce analytics reports.

```bash
/people-report <report type>
```

**Report types:** headcount, attrition, diversity, org health

## Standalone + Supercharged

Every command works without integrations, but connecting your tools unlocks automatic data:

| What You Can Do | Standalone | Supercharged With |
|-----------------|------------|-------------------|
| Draft offers | Provide details manually | HRIS, ATS for auto-fill |
| Onboarding checklists | Describe your process | HRIS, Knowledge base for templates |
| Performance reviews | Manual input | HRIS for review history |
| Policy lookup | Paste handbook content | Knowledge base |
| Comp analysis | Upload CSV, describe bands | Compensation data MCP |
| People reports | Upload data | HRIS for live data |

## Connectors

The plugin uses placeholders like `~~HRIS` and `~~ATS` for tool-agnostic workflows:

| Category | Placeholder | Included Servers | Other Options |
|----------|-------------|------------------|---------------|
| ATS | `~~ATS` | — | Greenhouse, Lever, Ashby, Workable |
| Calendar | `~~calendar` | Google Calendar | Microsoft 365 |
| Chat | `~~chat` | Slack | Microsoft Teams |
| Email | `~~email` | Gmail, Microsoft 365 | — |
| HRIS | `~~HRIS` | — | Workday, BambooHR, Rippling, Gusto |
| Knowledge base | `~~knowledge base` | Notion, Atlassian (Confluence) | Guru, Coda |
| Compensation data | `~~compensation data` | — | Pave, Radford, Levels.fyi |

## Settings

Create a local settings file at `human-resources/.claude/settings.local.json` to personalize:

```json
{
  "company": "Your Company",
  "headquarters": "City, State",
  "employeeCount": 500,
  "benefits": {
    "healthInsurance": "Provider Name",
    "pto": "Unlimited / X days",
    "parentalLeave": "X weeks"
  },
  "compensation": {
    "currency": "USD",
    "equityType": "RSU / Options",
    "vestingSchedule": "4 years, 1 year cliff"
  }
}
```

The plugin will ask you for this information interactively if it's not configured.

## Tips

1. **Include total comp** — Candidates compare total compensation, not just base salary
2. **Be specific about equity** — Share count, current valuation method, and vesting schedule
3. **Personalize offers** — Reference something from the interview process to make it warm
4. **Upload proposals** — For vendor analysis, upload the full proposal for detailed extraction
